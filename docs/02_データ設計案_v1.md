# データ設計案(v1.4)

作成日: 2026-07-09
更新日: 2026-07-16(v1.4: 予備数を廃止し、ロットへの割当を必要数のみに変更。予備は未割当=品番プールとして管理。「残必要数」を「未発送数」に改名)
更新履歴: v1.3 (2026-07-14) 搬入をロット無し記録に変更し、ロットへの割当を自動計算化。予備数列を追加 / v1.2 (2026-07-13) 納期先をロットIDに含める方針へ変更。中間状態「〜中」を追加
前提: [01_要件整理_v3.md](01_要件整理_v3.md)

## 設計方針

- **台帳(イベントログ)方式**: 在庫の増減・状態変化・移動をすべて 1 行 = 1 イベントとして記録する。
- **現在庫は台帳から集計して導出する**(スナップショットを別途持たない)。規模が小さい(品番 60 × 最大 30 個 × 週次)ため、SUMIFS/ピボットで都度計算して十分。
- データは UI(VBA / PowerShell+XAML)から独立した「ただのテーブル(シート)」として設計する。
- 入力ミスは行の直接修正・削除で訂正する(取消行方式は採らない)。

## 在庫の粒度

倉庫Bの在庫は次の 4 軸の内訳で把握する:

```
品番 × ロット(または未割当) × 処理段階(状態) × 数量
```

- **ロットID = 品番 + 出荷週(月曜日付) + 納期先** で一意。例: `ABC-123_2026-07-13_納期先A`(納期先が無い場合は `ABC-123_2026-07-13`)
- **納期先が異なればロットも分ける**(v1.2 で変更)。同一品番・同一週でも納期先Aと納期先Bは別ロットとして在庫・工程・充足判定を行う。納期先の一覧は M_リスト で管理する。
- ロットに紐付かない在庫は「**未割当(余剰)**」として品番単位で持つ。
- 状態: `未処理` / `アニール中` / `アニール済み` / `アセンブリ中` / `アセンブリ済み` / `梱包中` / `梱包済み` / `発送準備中` / `発送準備完了` / `不良`(「〜中」は開始イベントで入る作業中状態)

## テーブル 2: T_納期計画

納期一覧表 Excel から取り込んで作る。**1 行 = 品番 × 出荷週 × 納期先**(納期先が分かれる場合は複数行になる)。

| 列 | 内容 |
|---|---|
| ロットID | 品番 + "_" + 出荷週月曜日 + "_" + 納期先(納期先が空欄なら品番+週のみ。自動計算列) |
| 品番 | |
| 出荷週 | 月曜日の日付 |
| 納期先 | 出荷先(M_リストの納期先から選択。1 つしかない場合は空欄でも可) |
| 必要数 | 納期一覧表のセル値(納期先ごと) |
| 計画状態 | 有効 / キャンセル(再取込でセルが空になったらキャンセル扱い) |
| 最終取込日時 | |

- 主キーは 品番 × 出荷週 × 納期先 = ロットID(1 行 = 1 ロット)。
- 充足判定はロット(品番×週×納期先)ごとに行う(v1.2 で納期先合算をやめた)。

### 取込(再取込)の動き

納期変更・数量変更・キャンセルが「ちょくちょく」あるため、**取込は毎回全量スキャン+突合**とする:

1. 納期一覧表の 2 行目(E 列以降)から週の月曜日付を読む。
2. 6 行目以降の品番行 × 週列の交点で、数量が入っているセルを列挙する。
3. T_納期計画と突合:
   - 新しい 品番×週 → 行を追加
   - 数量が変わった → 必要数を更新
   - 表から消えた(セルが空になった) → 計画状態をキャンセルに変更(行は消さない。既に搬入済みの実績と突き合わせるため)

### イベント種類と在庫への影響

| イベント種類 | 在庫への影響(倉庫B) | 先状態 |
|---|---|---|
| 搬入(A→B) | **品番プールに +N(ロットIDは入力しない)** | 未処理 |
| アニール開始 | 元状態の在庫を N 個、状態変更 | アニール中 |
| アニール完了 | 同上 | アニール済み |
| アセンブリ開始 | 同上 | アセンブリ中 |
| アセンブリ完了 | 同上 | アセンブリ済み |
| 梱包開始 | 同上 | 梱包中 |
| 梱包完了 | 同上 | 梱包済み |
| 発送準備開始 | 同上 | 発送準備中 |
| 発送準備完了 | 同上 | 発送準備完了 |
| 発送 | ロットから −N(在庫から消える) | — |
| 不良発生 | 元状態の在庫を N 個、不良へ | 不良 |
| 廃棄 | 不良在庫から −N | — |
| 倉庫A戻し(B→A) | 対象在庫から −N(状態・元ロットは備考/元状態で残る) | — |
| 余剰化 | ロット → 未割当へ N 個振替(状態維持)※自動割当により通常は不要 | (維持) |
| 充当 | 未割当(またはロット)→ 充当先ロットへ N 個振替(状態維持) | (維持) |

### ロットへの自動割当(v1.3 で追加、v1.4 で必要数のみに変更)

- 搬入はロットを指定しない。品番ごとの「倉庫Bプール」= Σ搬入 −(ロット無しの発送・倉庫A戻し・不良)。
- プールから **出荷週の早い順(同週は計画の行順)** に、各ロットの **必要数まで**(キャンセルは 0)自動で割り当てる(V_充足確認の「割当数」列)。
- 割り当てられなかった残りが「未割当(余剰)」= 予備を含む品番単位の倉庫B在庫。**予備はロットに紐づけない**(2026-07-16 確定: 搬出・工程を進めるのは純粋に必要数のみ、という運用に合わせる。旧・予備数列は廃止)。
- 不良で補填が必要になったら「充当」イベントで未割当→ロットへ振り替える。
- 工程イベント(アニール開始・梱包完了など)は従来どおり **ロットIDを指定して** 記録する。ロットの割当数を超えて工程を記録すると V_在庫内訳 の「未処理」がマイナス表示になる(=充当が必要というサイン)。

### 「割当」イベントの扱い(2026-07-15 確定)

- **当面、台帳に「割当」イベントは記録しない。** 割当は倉庫内で物が動かない「計算結果」であり、台帳の原則は「人が行った物理的な作業の記録」。導出できるものを台帳にも書くと真実が2か所になり、納期・搬入の変更のたびに割当行の手修正が必要になるため。
- 「割当済みだが未着手」の状態は **V_在庫内訳 のロット行「未処理」列** で確認する。自動割当は搬入と同時に効くため、割当日=搬入日として履歴(搬入行〜開始行の間)からも追える。
- 例外的に明示割当したい場合(不良の補填など)は既存の **「充当」イベント** を使う(未割当→充当先ロット。台帳に履歴が残り、集計も追従する)。
- **UI化フェーズで、搬入登録時に割当行(充当行)を自動生成する方式へ切り替える。** これにより人手の負担なしに割当履歴が台帳に残る。切替時に V_ シートの数式割当(V_充足確認「割当数」列と V_在庫内訳の未処理の入り)をイベントベースの集計へ戻す。

- 「先状態」はイベント種類から一意に決まるため、列としては持たない。
- 元状態は状態変更イベントに限らず、**在庫を減らす/移すイベント(発送・廃棄・倉庫A戻し・余剰化・充当)でも記録する**。これにより「ロット×状態」別の在庫内訳が台帳だけから機械的に集計できる(廃棄の元状態は常に「不良」、発送は通常「発送準備完了」なので UI 側で自動補完すれば入力負担はない)。
- 状態遷移の妥当性(例: 梱包完了の元状態は、その品番の工程パターンの最終処理段階であること)は入力時に UI 側でチェックする。

## 導出ビュー(集計)

台帳から次を計算して表示する:

### V_在庫内訳(優先度 2: ステータス把握)

`品番 × ロット/未割当 × 状態` ごとの現在数量。台帳の全イベントを上表のルールで加減算した結果。

### V_充足確認(優先度 1)

有効な各ロット(T_納期計画)について:

```
割当数   = 品番プールから出荷週順に必要数まで自動割当した数(キャンセルは 0)
ロット在庫 = 割当数 + 充当受入 − 発送・倉庫A戻し・余剰化・不良・充当放出
未発送数 = MAX(0, 必要数 − 発送済数)(キャンセルは 0)
不足数   = MAX(0, 未発送数 − ロット在庫 − 同品番の未割当在庫)(正なら「要搬出」)
```

- 判定: キャンセル / 発送済(未発送数 = 0)/ 要搬出(不足数 > 0)/ OK。
- v1.4 で「残必要数」を「未発送数」に改名(予備が混ざらなくなり「必要数のうち未発送の個数」という意味が明確になったため)。
- 別ロットに割当済みの在庫はカウントしない(自動割当が排他的に配分するため自然に守られる)。
- 未割当在庫は同品番の複数ロットの判定に重複カウントされ得るが、通常は予備+端数程度で小さい。

## シート構成(Excel ブック「在庫管理台帳.xlsx」として実装済み)

| シート | 内容 | テーブル名 |
|---|---|---|
| 説明 | 入力ルール・クイックガイド(これをしたい→どこに入力) | — |
| 業務フロー | 状態(箱)とイベント(矢印)の流れ図(オートシェイプ) | — |
| V_充足確認 | 充足判定(数式・自動計算) | — |
| V_在庫内訳 | 状態別在庫の集計(数式・自動計算) | — |
| T_台帳 | イベントログ — 基本はここだけ日々入力 | tbl_ledger |
| T_納期計画 | 取込結果 | tbl_plan |
| M_品番 | 品番マスタ | tbl_master |
| M_リスト | 入力規則用リスト | — |

- シート名接頭辞: **V_ = View(自動計算・参照専用)/ T_ = Transaction(日々の入力データ)/ M_ = Master(基礎情報)**。
- テーブル名は `tbl_` 接頭辞+小文字始まりの命名(tbl_ledger 等)。
- **背景が灰色のセルは自動計算列**(V シート全体と T_納期計画のロットID列)。手入力しない。
